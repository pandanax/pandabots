# Работа с AI агентом и облаком

## Проблема (РЕШЕНА)

~~AI агент в Cursor не может напрямую выполнять команды в терминале из-за ограничения "Agent terminals are read-only".~~

**ОБНОВЛЕНИЕ:** Проблема решена! После настройки разрешений в Cursor, AI агент теперь может:
- Выполнять команды в терминале напрямую
- Получать результаты выполнения
- Работать с yc CLI автоматически

## Текущий подход

AI агент может работать с облаком автоматически через команды yc CLI.

## Workflow

1. **AI агент создает:**
   - Bash скрипты для операций
   - Python скрипты для сложных задач
   - Готовые команды для копирования

2. **Пользователь выполняет:**
   - Запускает скрипты в своем терминале
   - Копирует результаты обратно в чат
   - Подтверждает успешное выполнение

3. **AI агент обновляет:**
   - Документацию на основе результатов
   - Следующие скрипты/команды
   - Переходит к следующему шагу

## Примеры

### Проверка доступа к облаку

**Команда для выполнения:**
```bash
yc config profile activate sa-n8n-bot && yc compute instance list
```

### Использование wrapper скрипта

**Подготовка:**
```bash
chmod +x ./scripts/yc-wrapper.sh
```

**Использование:**
```bash
# Список виртуальных машин
./scripts/yc-wrapper.sh compute instance list

# Список подсетей
./scripts/yc-wrapper.sh vpc subnet list

# Информация о каталоге
./scripts/yc-wrapper.sh resource-manager folder get n8n
```

## Структура проекта

```
.
├── docs/              # Документация
├── scripts/           # Bash скрипты для операций
├── python/            # Python скрипты (когда потребуется)
└── .gitignore         # Исключения (credentials, secrets)
```

## Безопасность

- Все скрипты используют профиль `sa-n8n-bot`
- Авторизованные ключи не коммитятся в git
- Секреты хранятся в `~/.yc/` вне репозитория
